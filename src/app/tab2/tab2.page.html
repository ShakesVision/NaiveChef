Logic

Introduction

The app uses firebase as a backend. The basic CRUD operations are done by the Firestore API, which is a real time, efficient database.

CRUD

6 properties on the object to be saved, displayed, updated or deleted:
  1.  title:string //Title of the recipe
  2.  time:number //Time required to make a particular recipe
  3.  serves:number //Number of servings
  4.  difficulty:string //Level of difficulty: Easy, Medium, Hard
  5.  ingredients:Array //Array of ingredients
  6.  instructions:string //Step by step instructions to cook

Features

Basic

  1.  Live/Realtime updates
  2.  Type check: Won't accept non-numeric data for time,serves
  3.  Limits user to enter only 3 pre-defined values for difficulty
  4.  Initializes diff=Easy by default to save user's time to ADD
  5.  Converts comma-separated list of ingredients to Array and then writes to DB
  6.  Instructions are converted into list of steps

Stats

  1.  Total Entries updated in real time
  2.  Counts Unique ingredients: Iterate over all objects>properties>ingredients[Array], add values in a SET to find unique values
  3.  Other stats for every property, which could be found easily using SORT functions below

Sort, Filter and Search

  1.  Default sort by title
  2.  Sort by specific property: Sort function comparing each value with others
  3.  Search in any property: Dafault by title
  4.  Sort radio buttons work as a property selector for search function too
  5.  Search in real time in recipe text, title, time, ingredients etc.

Conclusion
